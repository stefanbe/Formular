Anleitung zur Erstellung eines Formulares

HTML und CSS Kenntnisse sind Nötig

Generelles:

  • Die Formulardateien müssen im Ordner "formular" des Plugins sein und müssen die Endung ".php" haben.
  • Der Name der Formulardateien ohne ".php" ist auch gleich der Parameter für den Plugin Platzhalter.
  • Fals es eine CSS Datei mit gleichen Namen im Ordner "formular" gibt wird sie Automatisch geladen.

Der Inhalt einer Formulardatei:

  • in der Ersten Zeile muss "<?php if(!defined('IS_CMS')) die(); ?>" Stehen.
  • Danach ist nur HTML Quelltext erlaubt.
  • Das HTML <form></form> darf nicht Benutzt werden da es vom Plugin Erstellt wird.
  • Diese HTML Formular Elemente werden für die E-Mail Verwendet:
    <input type="(text, radio, checkbox, submit, reset)" />
    <textarea></textarea>
    <select></select>
  • Besondere Elemente für die E-Mail:
    <mail_subject></mail_subject>
    <mail_txt></mail_txt>
    <mail_from></mail_from>
    <mail_to></mail_to>

Generelle Regeln für HTML Formular Elemente:

  • Gemerell darf der Inhalt der Atribute name="" und id="" nur Einmal vorkommen.
  • Außer bei den Elementen vom Type "radio" und "checkbox" wenn mann eine Gruppe bilden möchte ist der Inhalt von name="" gleich um die Gruppe bilden zu können.
  • Bei allen Elemnten ist das name="" Attriebut mit Inhalt Pflicht.
  • Bei den HTML Formular Elementen vom type="text" muss das Attriebut value="" Angegeben werden mit Lehren Inhalt.
  • Bei allen Anderen Formular Elementen Auser der "textarea" muss ein Attriebut value="" Angegeben werden mit entsprechenden Inhalt.
  • Bei einenm select HTML Formular Element was nicht das Attriebut "multiple" hat kann das Plugin nur dann Erkennen das nichts Selectiert wurde wenn das erste option Element kein Attriebut value="" hat z.B. <option>Wählen sie was aus</option>
  • Bei HTML Formular Elementen mit dem name="ASTALAVISTA(_Zahl)" solte man es den Spammern Erschweren in dem mann sie als Pflichtfeld definiert mit einem Glaubwürdigen Label und wenn möglich sie nicht direckt mittels CSS auf display:none zu setzen. Beispiel siehe contact.php

Inhalt aus der Sprachdatei Benutzen "lang/formular_(Language Einstellung des CMS).txt"

  • Alle Einträge können Verwendet werden.
  • Es dürfen auch da Platzhalter Verwendet werden z.B. "{DATUM}".
  • Auch eigene können Erstelt werden, sie müssen dann mit "usertext_" Anfangen.
  • Die Platzhalter werden aus dem Name des Eintrags Erstellt in dem der Anfang ("formular_" oder "usertext_") Entfernt wird und der Rest in Großbuchstaben Umgewandelt wird.
    Beispiele:
    formular_submit = {SUBMIT}
    usertext_user_mail = {USER_MAIL}
  • Achtung es macht nur sin die Sparachdatei zu nutzen bei mehrere Sprachen.

Besonderer Inhalt für das name="" Attriebut der HTML Formular Elemente:

type_mail Der Inhalt wird auf eine gültige E-Mail Adresse überprüft
type_mail_cc ist für ein HTML Formular Checkbox Element. Wenn das Häckchen gesetzt ist wird die Inhalt des HTML Formular Elements mit dem name="type_mail" Benutzt um eine Kopie der E-Mail an diese zu Senden.
Tip wenn immer eine Kopie gesendet werden soll:
Das HTML Element name="type_mail" zum Pflichtfeld machen.
Das HTML Element name="type_mail_cc" mit dem Attriebut checked="checked" versehen und es über CSS auf display:none setzen
type_title Wird nur für den Platzhalter {TITLE} Verwendet
type_forename Wird nur für den Platzhalter {FORENAME} Verwendet
type_name Wird nur für den Platzhalter {NAME} Verwendet
SUBMIT(_Zahl) Absende Button. Bei mehreren ist die Angabe (_Zahl) zu Vwerwenden z,B. SUBMIT_1, SUBMIT_2 u.s.w.
SPAM_TASK Für das Lösungs Element der Spam Frage. Die Spam Frage wird mit Platzhalter {SPAM_TASK} gestelt.
ASTALAVISTA_(Zahl) Irgend ein HTML Formular Elemente das wenn es Verändert wird den Absender als Spammer Erkent. Bei mehreren ist die Angabe (_Zahl) zu Vwerwenden z,B. ASTALAVISTA_1, ASTALAVISTA_2 u.s.w.

Vorgegebene HTML Classnamen für HTML Elemente:

Für Meldungs HTML Elemente:

fm-success-nodisplay Erfolgreich Versendet.
fm-mandatory-nodisplay nicht alle Pflicht Felder wurden Ausgefühlt.
fm-mailvalid-nodisplay E-Mail Adresse nicht gühltig.
fm-senttoofast-nodisplay Die Absende Wartezeit ist noch nicht Abgelaufen.
fm-mailcopy-nodisplay Es wurde keine E-Mail Adresse im HTML Formular Element name="type_mail" Eingetragen und das HTML Formular Element name="type_mail_cc" ist Aktiv deshalb kann keine Kopie Versendet werden.
fm-box-type Wird bei eintreten eines Ereignisses zu "fm-box-success" oder ".fm-box-error".
Kann Benutzt werden um alle Meldungen in einer Box unterzubringen.
Bei eintreten eines Ereignisses wird aus dem Entsprechenden Classnamen "-nodisplay" Entfernt.
Wenn einer dieser Classnamen nicht Verwendet wird gibts auch für dieses Ereigniss keine Meldung.
Beispiel:
  • ein Pflicht HTML Formular Element wurde nicht Ausgefühlt dann wird aus
    <div class="fm-mandatory-nodisplay">Pflicht Felder nicht Ausgefühlt</div>
  • das
    <div class="fm-mandatory">Pflicht Felder nicht Ausgefühlt</div>

Für HTML Elemente:

ein_class_name-mailnovalid Wenn im HTML Formular Element mit dem name="type_mail" keine gültige E-Mail Adresse Eingegeben wurde wird "-mailnovalid" aus allen Classnamen Entfernt.
fm-no-mtext HTML Formular Elemente die diesen Classname haben werden nicht im E-Mail Text Verwendet. Weiter Infos siehe "E-Mail".

Pflichtfelder Definieren:

Die Pflichtelemente werde über den Classnamen gesteuert.
Sobald im Classnamen der Text "-fm-(eine Zahl)-ma" enthalten ist es ein Pflichtelement.
Wenn so ein Element nicht Ausgefühlt oder Selectiert wird wird der Text "-fm-(eine Zahl)-ma" aus dem Classnamen entfernt und zwar bei allen die den gleichen Text "-fm-(eine Zahl)-ma" haben.
Beispiel:
<label id="test" class="labelclass-fm-1-ma">Namen</label>
<input type="text" name="test" class="inputclass-fm-1-ma" />
Bei nicht Ausfühlen wird daraus
<label id="test" class="labelclass">Namen</label>
<input type="text" name="test" class="inputclass" />
So kann jetzt die Classen Namen "labelclass" und "inputclass" für die Kentlich machung des nicht Ausgefühlten Elements Verwendet werden.
Für beide kann natürlich auch nur ein Classname Verwendet werden.
<label id="test" class="eineclasse-fm-1-ma">Namen</label>
<input type="text" name="test" class="eineclasse-fm-1-ma" />

Platzhalter die im HTML Formular Verwendet werden:

{URL_SRC} Der Url Pfad in den Ordner "formular" z.B. für ein Bild oder Javascriptdatei
{SPAM_TASK} Die Spam Frage zu dem HTML Formular Elementen mit dem name="SPAM_TASK".

Plugin Eigene Elemente für die E-Mail:

Pflicht Elemente:

<mail_subject></mail_subject> Der E-Mail Betreff.
<mail_txt></mail_txt> Der E-Mail Text. Enthaltene Zeilenumbrüche, Lehrzeichen und Tabs werden Verwendet.

Optinale E-Mail Adressen Elemente:

Wenn Angegeben werden diese Benutzt. Es gelten die gleichen Regeln wie in den Plugin Einstellungen.
<mail_from></mail_from> Absender E-Mail Adresse (eine).
<mail_to></mail_to> Empfänger E-Mail Adressen (mehrere).

Eine Selectbox für die Empfänger im Formular

Die Selectbox muss ein name Attriebut haben mit dem Inhalt mail_to
und die Option ein value mit einer Fortlaufender Nr. die auf die E-Mails im Plugin (Empfänger E-Mail Adressen.) oder <mail_to></mail_to> Zeigen.

Beispiel:

<select name="mail_to">
<option value="1">Demo Empfänger 1</option>
<option value="2">Demo Empfänger 2</option>
<option value="3">Demo Empfänger 3</option>
</select>
<mail_to>demo1@mail.de, demo2@mail.de, demo3@mail.de</mail_to>

Text und Select Elemente werden nur mit Titel Angezeigt wenn sie Ausgefühlt wurden.

Titel Steuerung:

Bei den Elementen "input" vom Type "text" und "textarea" und "select"

Wird als Titel das "label" Element verwendet und als Inhalt die "value" oder was zwischen <option>Das hier</option> steht

Bei den Elementen "input" vom Type "radio" und "checkbox"

Wird das "label" Element als Antword genommen wenn forhanden ansonsten der inhalt von "value".
und als Titel wenn Vorhanden das Element mit der id="title_for_(name Attriebut)".

Classname "fm-no-mtext":

Auf ein Elementen "input", "textarea" und "select" Angewende.
Erscheint es mit seinem Titel nicht im Mail Text.
Auf einem "label" und id="title_for_(name Attriebut)" Angewende.
Erscheint nur der Titel nicht nicht im Mail Text.
Wenn ein HTML Element mit dem Classname "fm-no-mtext" Innerhalb eines "label" oder id="title_for_(name Attriebut)" Elementes ist.
Wird es Daraus Entfernt.
Beispiel:
<label for="Text">Name<span class="fm-no-mtext"> ist Pflicht</span></label>
Ergebnis im Mailtext = Name
Bei HTML Formular Elemente vom Type "type_title", "type_forename" und "type_name" sind sie über die Platzhalter {TITLE}, {FORENAME} und {NAME} Verwendbar.

Platzhalter die verwendet werden:

{MAIL_TEXT} Der aus den Elementen generiete Text ist Pflicht.
{DATUM} das Aktuelle Datum.
{TITLE} Der Inhalt aus dem HTML Formular Element mit dem Namen "type_title"
{FORENAME} Der Inhalt aus dem HTML Formular Element mit dem Namen "type_forename"
{NAME} Der Inhalt aus dem HTML Formular Element mit dem Namen "type_name"
